🎯 Aristotle Dashboard

Setup guide — follow these steps once and you're done forever.

What you'll end up with

  • A dashboard at https://[your-username].github.io/aristotle-dashboard
  • Code runs automatically every weekday at 4:30 PM ET (after market close)
  • Shows today's signals + 30 days of history
  • Works on your iPhone from anywhere, no laptop needed

Part 1 — Create a GitHub Account

Step 1
Go to github.com and click Sign up. Use any email. Your username will appear in your dashboard URL — something like josephnunes or joe-nunes is fine.
Step 2
Once signed in, click the + icon (top right) → New repository.
  • Name it: aristotle-dashboard
  • Set to Public (required for free GitHub Pages)
  • Leave everything else as default
  • Click Create repository

Part 2 — Install Git on your Mac

Step 3
Open Terminal (search "Terminal" in Spotlight). Run: 
git --version
If it prints a version number, Git is already installed — skip to Step 4.

If not, macOS will prompt you to install Xcode Command Line Tools. Click Install and wait (~5 mins).
Step 4
Configure Git with your name and email (one-time setup): 
git config --global user.name "Joseph Nunes"
git config --global user.email "your-github-email@example.com"

Part 3 — Upload the files

Step 5
Copy the aristotle-dashboard folder (from the files delivered to you) to your Documents folder. You should have this structure: 
aristotle-dashboard/
├── Aristotle_v2.py
├── history.json
├── index.html
├── README.md
└── .github/
    └── workflows/
        └── run_aristotle.yml
⚠️ The .github folder starts with a dot — make sure it's included. On Mac, press Cmd+Shift+. in Finder to show hidden files.
Step 6
In Terminal, navigate to the folder and push it to GitHub. Replace josephnunes with your actual GitHub username: 
cd ~/Documents/aristotle-dashboard
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/josephnunes/aristotle-dashboard.git
git push -u origin main
GitHub will ask for your username and password. For the password, use a Personal Access Token (see Step 7).
Step 7 — GitHub Personal Access Token
GitHub no longer accepts your account password for Git. You need a token:
  • Go to github.com/settings/tokens/new
  • Note: Aristotle push
  • Expiration: No expiration
  • Tick repo scope
  • Click Generate token
  • Copy the token (starts with ghp_) — you only see it once
Use this token as the "password" when Terminal asks during Step 6.

Part 4 — Add your email credentials as Secrets

Step 8
Your Gmail credentials need to be stored securely in GitHub (never in the code):
  • Go to your repo on GitHub → Settings → Secrets and variables → Actions
  • Click New repository secret
  • Add two secrets:
Name: SMTP_USERNAME
Value: reactive.saturday@gmail.com

Name: SMTP_PASSWORD
Value: pnjt mzck ktdg sflw

Part 5 — Enable GitHub Pages

Step 9
  • In your repo → Settings → Pages
  • Under Source, select Deploy from a branch
  • Branch: main, folder: / (root)
  • Click Save
After ~60 seconds your site will be live at:
https://[your-username].github.io/aristotle-dashboard

Part 6 — Test it manually

Step 10
Don't wait until 4:30 PM ET — trigger a manual run now to check everything works:
  • Go to your repo → Actions tab
  • Click Run Aristotle Daily Scanner (left sidebar)
  • Click Run workflow → Run workflow
  • Watch the run — it takes about 3–5 minutes
  • When it shows a green tick, visit your GitHub Pages URL

Part 7 — Add to iPhone home screen

Step 11
On your iPhone:
  • Open Safari and go to your dashboard URL
  • Tap the Share button (box with arrow)
  • Tap Add to Home Screen
  • Name it Aristotle → tap Add
It now sits on your home screen like an app. Tap it anytime to see the latest signals.
✅ That's it. The scanner will now run every weekday at 4:30 PM ET automatically, update your dashboard, and you'll be able to check it from anywhere on your iPhone.

Ongoing workflow

  • Each day after US market close, GitHub runs the script automatically
  • Tap the home screen icon to see today's signals
  • Swipe to the History tab to see the last 30 days
  • To update the strategy or dashboard design, edit the files and push: git add . && git commit -m "update" && git push
